1          Introducción

 

Este documento describe los estándares y nomenclaturas que se utilizarán en la definición y elaboración de las estructuras de servidores, particiones, procesos, archivos, directorios y carpetas, para los diferentes productos y Software's que conformarán los aplicativos del facturador dentro del proyecto Programa  Atención Total de CANTV.

2          Nomenclatura y terminología

 

Cada estructura consta de una serie de letras y números (identificadores) que en función de su ubicación definen los distintos servidores, ambientes, bases de datos,  archivos, desarrollos, procesos,etc.

A continuación se describen los identificadores que se utilizarán en la definición de nombres.

4          Sección para Aplicativos y Bases de Datos

4.1         Definición Hostname del Servidor

 

Consiste en un identificador de 7 caracteres donde se inicia con un prefijo fijo de tres caracteres, seguido con dos caracteres que define el sistema (ST) al cual pertenece, culminando con un número consecutivo de dos caracteres asociado con un rango del aplicativo y el tipo de ambiente.

 

Plantilla:

SRV [ST] [RH]

 

Argumento

Significado

SRV

Prefijo indicativo para el nombre de servidor

ST

Define el sistema, ver punto 3.2, Tabla 2.

RH

Define el rango para el hostname de los servidores. Ver punto 3.11. Tabla 11.

 

Ejemplo:

SRVFT01

Nombre del servidor uno del Sistema de Facturación. 

 

 

4.2         Definición del nombre del Ambiente

 

Consiste en un identificador de 7 caracteres, donde se define la aplicación, el ambiente y un número secuencial que identifica el dominio.

 

Plantilla:

[AP] [AB] nn

 

Argumento

Significado

AP

Identifica la aplicación del sistema, Ver punto 3.4,Tabla 4.

AB

Identifica el nombre del ambiente. Ver punto 3.1, Tabla 1.

nn

Es un número secuencial que identifica al ambiente.

 

Ejemplo:

KBPD01

Identifica el ambiente 01 Desarrollo de Kenan

 

4.3         Definición del nombre de la Base de Datos

 

Consiste en un identificador con un máximo de 8 caracteres, en él se incluye la aplicación, ambiente, un número consecutivo y el tipo de base de datos. Para el aplicativo de Kenan el argumento AP se identifica con un carácter fijo K y para el argumento N se define por dos caracteres consecutivos. El resto de los aplicativos el argumento TB está identificado solo por ADM, para Kenan este argumento corresponde a ADM, CAT, CUS1, CUS2 y CUS3.

 

Plantilla:

[AP] [AB] N [TB]

 

Argumento

Significado

AP

A excepción de la aplicación de Kenan, Ver punto 3.4,Tabla 4. Para Kenan corresponde a un carácter fijo K.

AB

Identifica el nombre del Ambiente. Ver punto 3.1,Tabla 1.

N

Es un número secuencial que identifica al ambiente. Para la aplicación de Kenan el número secuencial contiene dos caracteres consecutivos (nn).  

TB

Se refiere a la base de datos.  Ver punto 3.5,Tabla 5.

 

Ejemplo:

KD01CUS1

Identifica la base de datos de desarrollo uno de Kenan customer 1

 

Ejemplo:

CDMP1ADM

Identifica la base de datos uno de producciòn de CDM

11.8     Condiciones de Entrada  Salida (Globales)

 

Las condiciones globales son aquellas que se definen dentro de la herramienta y permiten la comunicación entre distintas plataformas. Ejemplo: Mainframe/Unix  Unix/Mainframe.

11.9     Usuario de Ejecución

 

La herramineta Control-M requiere un usuario que permita la ejecución de los distintos procesos el cual esta definido con el nombre de  ctmft . Este usuario es creado bajo el ambiente unix con   la permisologia adecuada para poder realizar las diferentes actividades, por ejemplo: ejecución, escritura, creación,etc.
